Création dynamique de contrôle

Au lieu de créer des contrôles graphiques dans la phase de conception du programme, ceux-ci peuvent être insérés dynamiquement par le programme au moment de l'exécution de ce programme. Il est évident que si nous souhaitons créer un lot de contrôle du même type par exemple 10 boutons, le plus simple est de faire un tableau de 10 boutons, afin d'obtenir une interface graphique semblable à celle-ci :


  1. Où mettre le code de création ?
Il faut que ce code s'exécute au chargement de notre fenêtre afin que celle-ci apparaisse complète sans intervention de l'utilisateur. Notre code va donc se trouver en début de programme après la phase d'initialisation des composants :

Pour que notre tableau de boutons soit visible dans toute l'application nous allons le déclarer juste après la déclaration du cadre (frame) principal.

  1. Que doit faire ce code ?
  1. créer le contrôle
  2. donner sa position sur la frame
  3. lui donner une taille
  4. mettre un texte si nécessaire
  5. donner le nom de la méthode à exécuter en cas d'action sur le bouton
  6. l'ajouter à la frame
  7. demander l'affichage de la feuille modifiée
  1. Ecrire la méthode qui s'exécute en cas d'action sur le bouton
Nous savons que lorsqu'un clic de souris est effectué il est pris en charge par le système qui répère l'application concernée sur l'écran, lors d'un clic sur un bouton de la fenêtre de notre programme, l'événement sera donc transmis à la Java Machine Virtuelle, qui retransmet cet événement à notre programme, il suffit donc de signaler dans notre programme java que nous souhaitons récupérer l'événement lors qu'une action sur un bouton est effectuée. En terme technique on dit implémenter un ActionListener (un "auditeur d'action" !). C'est ce que nous allons implanter.

Quel que soit le bouton actionné, c'est la même méthode qui va s'exécuter, il faut donc que lors de l'appel de cette méthode nous sachions quel bouton a été cliqué, on dit en terme technique récupérer l'adresse du contrôle appelant, nous pourrons alors effectuer l'action souhaité par le programme.
  1. récupération de l'adresse
  2. actions à effectuer

EXEMPLE

Créons un petit programme qui génère 10 boutons numérotés de 1 à 10, qui aura le comportement suivant : lorsqu'un bouton est cliqué il devient grisé et n'est plus actif (il n'est plus enabled), de plus le programme affiche dans la fenêtre d'exécution le numéro du bouton lorsqu'il est cliqué.

Il nous faut en premier lieu déclarer un tableau de bouton. Bien évidemment Netbeans vous demande d'importer une bibliothèque pour les boutons, et vous acceptez.

Voici l'image du code :

Voici le source ce qui vous permet de faire un copié/collé dans votre code :

// Les variables horiz et vert permettent de positionner les boutons
// c'est le point de départ du premier bouton
int Horiz, Vert;
Horiz = 60;
Vert = 100;
// On crée les 10 boutons
for (int i = 1; i < 10; i++) {
// Création du bouton
lesBoutons[i] = new JButton();
// Emplacement
lesBoutons[i].setLocation(Horiz, Vert);
// Taille
lesBoutons[i].setSize(50, 50);
// Numéro
lesBoutons[i].setText(String.valueOf(i));
// ajout de l'action en cas de pression
lesBoutons[i].addActionListener(new MyActionListener());
// on ajoute les chiffres au panel
mainPanel.add(lesBoutons[i]);
Horiz = Horiz + 60;
mainPanel.repaint();
}

Vous remarquez que Netbeans vous indique que la class MyActionListener n'existe pas, patience nous allons l'écrire !

Entrez la définition de classe suivante avec ses crochets en fin de fichier juste avant les déclarations, regardez où est l'ascenseur à droite et positionnez-vous au même endroit.

private class MyActionListener implements ActionListener {
}

Bien évidemment, Netbeans n'est pas satisfait, il souhaite créer toutes les méthodes associées à une action :

Laissons le faire et il nous crée le code suivant :

Nous pouvons nous débarrasser du code inutile, c'est à dire du code de la méthode proposé par défaut et entrer le code qui permettra le traitement que nous souhaitons effectuer :

public void actionPerformed(ActionEvent e) {
JButton courant = (JButton) e.getSource();
System.out.println("Enabled " + courant.getText());
courant.setEnabled(false);
}
